## 1. 首页（双层结构视图）

### 1.1 页面目标

- 核心功能：展示"历史/逾期区 + 主日期流矩阵"的双层结构
- 用户价值：一眼看清逾期事项、历史记录、未来任务分布
- 关键体验：逾期事项突出显示、历史记录可折叠、主日期流流畅滚动

### 1.2 页面布局

#### 整体结构（双层布局）


### 1.4 用户操作路径

#### 路径 1：浏览任务

```
进入首页
  ↓
默认展示：逾期区（展开）+ 历史区（折叠）+ 主日期流（今天起30天）
  ↓
上下滑动浏览日期
  ↓
左右滑动查看全部分类
  ↓
点击任务卡片查看详情
```

#### 路径 2：查看逾期事项

```
进入首页
  ↓
逾期区默认展开，显示所有逾期未完成事项
  ↓
点击逾期事项进入详情
  ↓
或点击逾期区标题折叠该区域
```

#### 路径 3：查看历史事项

```
进入首页
  ↓
点击"历史事项"标题展开历史区
  ↓
查看所有历史已完成事项
  ↓
点击历史事项进入详情
```

#### 路径 4：切换视图模式（Alpha 后段）

```
点击左上角 ⇄ 按钮
  ↓
弹出确认对话框："切换为分类纵向视图？"
  ↓
确认后重新渲染
  ↓
原日期列变为横向，分类列变为纵向
```

#### 路径 5：快速定位今天

```
点击"回到今天"按钮
  ↓
平滑滚动动画回到今天所在行
  ↓
今天行高亮闪烁一次（300ms）
```

### 1.5 逾期区组件规格

#### 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ 逾期事项 (N)                                  [折叠按钮] │ │  ← 标题栏
│ └─────────────────────────────────────────────────────────┘ │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ ┌─────────────────────────────────────────────────────┐ │ │
│ │ │ 🔴 任务标题（逾期N天）              分类名称        │ │ │  ← 任务卡片
│ │ └─────────────────────────────────────────────────────┘ │ │
│ │ ┌─────────────────────────────────────────────────────┐ │ │
│ │ │ 🔴 任务标题（逾期N天）              分类名称        │ │ │
│ │ └─────────────────────────────────────────────────────┘ │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

#### 组件属性

| 属性 | 值 |
|------|-----|
| 默认状态 | 展开 |
| 标题 | "逾期事项 (N)"，N 为逾期任务数量 |
| 折叠按钮 | ▼（展开时）/ ▶（折叠时） |
| 空状态 | 不显示该区域 |

#### 任务卡片样式

| 元素 | 样式 |
|------|------|
| 卡片背景 | 浅红色 #FFEBEE |
| 卡片边框 | 左侧红色条 4px |
| 任务标题 | 红色 #F44336，加粗 |
| 逾期标签 | "逾期 N 天"，红色小标签 |
| 分类名称 | 灰色，右侧对齐 |

#### 排序规则

1. 按逾期天数倒序（逾期越久越前）
2. 同逾期天数按优先级排序（高 > 中 > 低）
3. 同优先级按截止时间排序

### 1.7 主日期流矩阵规格

#### 日期列固定方式

**方式：Sticky Header（推荐）**

```dart
custom_scroll_view(
  slivers: [
    sliver_to_box_adapter(
      position: sticky_left,
      child: date_column(),
    ),
    sliver_list(
      scroll_direction: horizontal,
      children: category_columns(),
    ),

```

#### 滚动行为

| 操作 | 效果 |
|------|------|
| 上下滑动 | 整个页面滚动（固定区域 + 主日期流） |
| 左右滑动 | 仅分类列横向滚动，日期列保持固定 |
| 惯性滚动 | 支持，滚动停止时无对齐要求 |
| 快速滚动 | 显示滚动指示器（Scrollbar） |

#### 分类表头滚动方式

- 表头区域和内容区域使用同一个 ScrollController
- 左右滑动时，表头和内容同步滚动
- 滚动位置保持同步，无错位

### 1.8 单元格任务展示规则

#### 单元格尺寸

| 属性 | 值 | 说明 |
|------|-----|------|
| 宽度 | 120dp | 固定宽度 |
| 最小高度 | 80dp | 无任务时的高度 |
| 最大高度 | 无限制 | 随任务数量扩展 |

#### 任务卡片尺寸

| 属性 | 值 |
|------|-----|
| 宽度 | 112dp（单元格宽度 - 8dp margin） |
| 高度 | 自适应，最小 32dp |
| 外边距 | 4dp |
| 圆角 | 8dp |

#### 任务展示规则

**截止前3天开始显示**：

```
假设任务截止日：3月20日

3月17日：开始显示（截止前3天）
3月18日：显示
3月19日：显示
3月20日：高亮显示（截止日）
3月21日：不显示（已逾期，进入逾期区）
```

**任务排序规则**：

1. 未完成在前，已完成在后
2. 未完成中：优先级高 → 中 → 低
3. 同优先级按截止时间排序（有时间的在前）
4. 无截止时间的按创建时间排序

**任务卡片内容**：

```
┌─────────────────────────┐
│ ● 任务标题（最多2行）     │  ← 优先级圆点 + 标题
│    14:00               │  ← 截止时间（如有）
└─────────────────────────┘
```

### 1.9 XY 轴互换前后布局对比（Alpha 后段）

#### 模式 A → 模式 B 的映射关系

| 元素 | 模式 A（默认） | 模式 B（互换后） |
|------|---------------|-----------------|
| 纵向主轴 | 日期 | 分类 |
| 横向副轴 | 分类 | 日期 |
| 固定列 | 日期列 | 分类列 |
| 滚动区域 | 分类列 | 日期列 |
| 单元格内容 | 某日期某分类的任务 | 某分类某日期的任务（相同） |

#### 互换动画

- 过渡动画：300ms ease-in-out
- 动画效果：淡出旧布局 → 淡入新布局
- 保持滚动位置百分比（如原来滚动到50%，新布局也在50%）

### 1.10 分类过多/任务过多时的处理

#### 分类过多（> 5个）

**处理方案**：
- 分类列横向滚动（始终）
- 表头显示滚动指示器（右侧渐变遮罩）
- 显示"更多"提示：右侧显示半透明白色遮罩 + 箭头

```
┌────────┬─────────┬─────────┬─────────┬─────────┬───────┐
│ 日期   │ 分类A   │ 分类B   │ 分类C   │ 分类D   │ ▶     │  ← 右侧箭头提示
├────────┼─────────┼─────────┼─────────┼─────────┼───────┤
```

#### 任务过多（单个单元格 > 5个）

**处理方案**：
- 单元格内任务列表可滚动
- 或显示"+3 更多"，点击进入该日期/分类的详情视图
- Alpha 推荐：单元格内滚动

```
┌─────────────────┐
│ 任务1           │
│ 任务2           │
│ 任务3           │
│ 任务4           │
│ 任务5           │
│ +3 更多...      │  ← 点击展开或跳转
└─────────────────┘
```

#### 日期范围固定

- Alpha 阶段主日期流固定展示"今天起未来 30 天"
- 不支持时间范围筛选
- 过去日期通过逾期区/历史区承接

### 1.11 空状态

#### 场景 1：逾期区无任务

- 不显示逾期区区域

#### 场景 2：历史区无任务

- 不显示历史区区域
- 直接显示主日期流

#### 场景 3：逾期区和历史区都无任务

- 不显示固定区域
- 直接显示主日期流

#### 场景 4：主日期流无任务

```
┌─────────────────────────────────────────┐
│                                         │
│                                         │
│              📋                         │
│                                         │
│         还没有任何任务                   │
│                                         │
│    点击右下角 + 按钮创建第一个任务        │

└─────────────────────────────────────────┘
```

- 全屏居中显示
- 图标：待办清单图标

### 1.12 错误状态

#### 错误 1：数据加载失败

```
┌─────────────────────────────────────────┐
│                                         │
│              ⚠️                         │
│                                         │
│         数据加载失败                     │
│                                         │
│      [ 重试 ]                           │
│                                         │
└─────────────────────────────────────────┘
```

- 显示错误图标 + 提示文字
- 提供重试按钮
- 保留本地缓存数据（如有）

#### 错误 2：滚动性能卡顿

- 检测帧率 < 30fps 时自动降级：
  - 隐藏任务卡片动画
  - 减少阴影效果
  - 使用更简单的布局

### 1.13 边界状态

| 场景 | 处理 |
|------|------|
| 分类数量为 0 | 显示引导创建分类的提示 |
| 日期跨月 | 显示月份分隔线 |
| 过去日期事项 | 进入逾期区（未完成）或历史区（已完成） |
| 逾期任务 | 红色边框 + 红色文字 + 逾期天数标签 |
| 已完成任务 | 文字划线 + 透明度 50% + 置底 |
| 超长任务标题 | 最多2行，超出显示... |
| 系统"未分类"分类 | 灰色标识，不可删除，不可编辑 |

### 1.14 验收标准

| 检查项 | 标准 |
|--------|------|
| 首页加载 | 30天内数据 2 秒内加载完成 |
| 滚动流畅度 | 60fps，无卡顿 |
| 逾期区显示 | 正确显示逾期事项，排序正确 |
| 历史区显示 | 默认折叠，展开后显示正确 |
| 轴互换（Alpha 后段） | 切换后 500ms 内完成渲染 |
| 分类过多 | 支持横向滑动，无布局错位 |
| 任务过多 | 单元格内可滚动，或显示"更多" |
| 空状态 | 所有空状态都有引导提示 |
| 响应式 | 适配不同屏幕宽度 |

## 2. 添加事项弹窗

### 2.1 页面目标

- 提供手动和 AI 两种添加任务的方式
- 降低任务录入成本
- 确保数据完整性

### 2.2 页面布局

```

│  添加事项                          [×]  │  ← 标题栏

├─────────────────────────────────────────┤
│                                         │
│              内容区域                    │  ← 根据 Tab 变化
│                                         │
├─────────────────────────────────────────┤
│         [ 取消 ]    [ 创建 ]             │  ← 底部按钮
└─────────────────────────────────────────┘
```

### 2.3 组件清单

| 组件名 | 类型 | 说明 |
|--------|------|------|
