## 1. 数据实体定义

### 1.1 分类（Category）

#### 字段定义

| 字段        | 类型       | 必填 | 默认值   | 说明          |
| --------- | -------- | -- | ----- | ----------- |
| id        | String   | 是  | UUID  | 唯一标识        |
| name      | String   | 是  | -     | 分类名称，最多10字符 |
| color     | String   | 是  | 随机    | 颜色值，HEX格式   |
| sortOrder | int      | 否  | 0     | 排序权重        |
| isSystem  | bool     | 否  | false | 是否系统内置分类    |
| createdAt | DateTime | 是  | now   | 创建时间        |
| updatedAt | DateTime | 是  | now   | 更新时间        |

#### 系统内置分类

| id            | name | color   | isSystem | 说明                 |
| ------------- | ---- | ------- | -------- | ------------------ |
| uncategorized | 未分类  | #9E9E9E | true     | 系统内置，不可删除/重命名/修改颜色 |

#### 约束规则

1. **名称唯一性**：同一用户下分类名称不区分大小写唯一
2. **系统分类保护**：`isSystem = true` 的分类不可删除、不可重命名、不可修改颜色
3. **删除迁移**：删除用户分类时，该分类下所有事项迁移到"未分类"
4. **默认分类**：创建事项时，分类默认值为"未分类"

### 1.2 事项（Item）

#### 字段定义

| 字段          | 类型        | 必填 | 默认值           | 说明                   |
| ----------- | --------- | -- | ------------- | -------------------- |
| id          | String    | 是  | UUID          | 唯一标识                 |
| title       | String    | 是  | -             | 任务标题，最多100字符         |
| description | String    | 否  | null          | 任务描述                 |
| categoryId  | String    | 是  | uncategorized | 所属分类ID               |
| dueDate     | DateTime  | 是  | -             | 截止日期（仅日期部分）          |
| dueTime     | TimeOfDay | 否  | null          | 截止时间（仅时间部分）          |
| priority    | Enum      | 否  | medium        | 优先级：high/medium/low  |
| status      | Enum      | 是  | pending       | 状态：pending/completed |
| imagePath   | String    | 否  | null          | 图片附件本地路径             |
| createdAt   | DateTime  | 是  | now           | 创建时间                 |
| updatedAt   | DateTime  | 是  | now           | 更新时间                 |
| completedAt | DateTime  | 否  | null          | 完成时间                 |

#### 重要说明

1. **status 字段只有两种值**：`pending`（未完成）和 `completed`（已完成）
2. **不存在** **`overdue`** **状态字段**：逾期是运行时计算状态，不持久化
3. **categoryId 默认值**：创建事项时如未指定分类，默认为 `uncategorized`
4. **dueDate 允许过去日期**：用于补录历史事项或记录逾期任务

### 1.3 计算属性（不存储）

| 属性          | 类型   | 计算方式                                       | 说明   |
| ----------- | ---- | ------------------------------------------ | ---- |
| isOverdue   | bool | `(status == pending) && (dueDate < today)` | 是否逾期 |
| displayArea | Enum | 根据 dueDate 和 status 判断                     | 展示区域 |
| overdueDays | int  | `today.difference(dueDate).inDays`         | 逾期天数 |

#### displayArea 计算规则

```dart
enum DisplayArea { mainStream, overdue, history }

DisplayArea get displayArea {
  if (dueDate.isBefore(DateUtils.today)) {
    return status == ItemStatus.completed 
        ? DisplayArea.history 
        : DisplayArea.overdue;
  }
  return DisplayArea.mainStream;
}
```

## 2. 状态流转规则

### 2.1 事项状态流转

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   创建事项                                               │
│       ↓                                                 │
│   ┌─────────┐                                          │
│   │ pending │ ←──────────────────────┐                 │
│   │ (未完成) │                        │                 │
│   └────┬────┘                         │                 │
│        │                              │                 │
│        │ 点击"完成"                    │ 点击"取消完成"   │
│        │                              │                 │
│        ↓                              │                 │
│   ┌───────────┐                       │                 │
│   │ completed │ ──────────────────────┘                 │
│   │ (已完成)   │                                         │
│   └───────────┘                                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**重要说明**：

- 数据库只存储 `pending` 和 `completed` 两种状态
- `overdue`（逾期）是运行时计算状态，不存储
- 状态变更时更新 `updatedAt` 和 `completedAt` 字段

### 2.2 逾期状态判定

**判定公式**：

```dart
bool get isOverdue {
  return status == ItemStatus.pending && 
         dueDate.isBefore(DateUtils.today);
}
```

**判定条件**：

1. 状态为 `pending`（未完成）
2. 截止日期早于今天（不含今天）

**重要说明**：

- 逾期状态**不持久化**
- 每次查询时实时计算
- 不存在"从逾期恢复"的概念，只有"完成逾期任务"或"修改截止日期"

### 2.3 展示区域判定

**判定规则**：

| dueDate | status    | displayArea | 说明         |
| ------- | --------- | ----------- | ---------- |
| < 今天    | pending   | overdue     | 逾期区        |
| < 今天    | completed | history     | 历史区        |
| >= 今天   | pending   | mainStream  | 主日期流       |
| >= 今天   | completed | mainStream  | 主日期流（置底显示） |

**代码实现**：

```dart
DisplayArea get displayArea {
  final today = DateUtils.today;
  
  if (dueDate.isBefore(today)) {
    return status == ItemStatus.completed 
        ? DisplayArea.history 
        : DisplayArea.overdue;
  }
  
  return DisplayArea.mainStream;
}
```

### 2.4 状态变更后的区域迁移

| 操作              | 原状态       | 新状态       | 原区域        | 新区域            |
| --------------- | --------- | --------- | ---------- | -------------- |
| 完成逾期事项          | pending   | completed | overdue    | history        |
| 取消完成历史事项        | completed | pending   | history    | overdue        |
| 修改截止日期到未来（逾期事项） | pending   | pending   | overdue    | mainStream     |
| 修改截止日期到过去（未完成）  | pending   | pending   | mainStream | overdue        |
| 完成未来事项          | pending   | completed | mainStream | mainStream（置底） |

## 3. 首页展示规则

### 3.1 首页双层结构

**整体布局**：

```
┌─────────────────────────────────────────┐
│           顶部导航栏（固定）              │
├─────────────────────────────────────────┤
│  ┌───────────────────────────────────┐  │
│  │     逾期区（固定区域）              │  │  ← 未完成 && dueDate < today
│  ├───────────────────────────────────┤  │
│  │     历史区（固定区域）              │  │  ← 已完成 && dueDate < today
│  └───────────────────────────────────┘  │
├─────────────────────────────────────────┤
│                                         │
│         主日期流矩阵（可滚动）            │  │  ← dueDate >= today
│         展示今天起未来30天               │  │
│                                         │
└─────────────────────────────────────────┘
```

### 3.2 逾期区规则

#### 显示条件

- 存在 `status == pending && dueDate < today` 的事项

#### 默认状态

- 展开

#### 排序规则

1. 按逾期天数倒序（逾期越久越前）
2. 同逾期天数按优先级排序（high > medium > low）
3. 同优先级按截止时间排序（有时间的在前）

#### 空状态

- 无逾期事项时不显示该区域

#### 样式规则

| 元素   | 样式             |
| ---- | -------------- |
| 卡片背景 | 浅红色 #FFEBEE    |
| 卡片边框 | 左侧红色条 4px      |
| 任务标题 | 红色 #F44336，加粗  |
| 逾期标签 | "逾期 N 天"，红色小标签 |

### 3.3 历史区规则

#### 显示条件

- 存在 `status == completed && dueDate < today` 的事项

#### 默认状态

- 折叠

#### 排序规则（展开后）

- 按完成时间倒序（最近完成的在前）

#### 空状态

- 无历史事项时不显示该区域

#### 样式规则

| 元素   | 样式             |
| ---- | -------------- |
| 卡片背景 | 白色             |
| 任务标题 | 中划线，透明度 50%    |
| 完成时间 | "N月N日完成"，灰色小标签 |

### 3.4 主日期流规则

#### 展示范围

- 起始日期：今天
- 展示天数：固定 30 天
- **Alpha 阶段不支持时间范围筛选**

#### 任务展示规则

**跨天展示**：

- 截止前 3 天开始在该日期显示该任务
- 截止日当天高亮显示
- 逾期后进入逾期区，不在主日期流显示

**示例**：

```
任务截止日：3月20日

展示日期：
- 3月17日：显示（截止前3天）
- 3月18日：显示
- 3月19日：显示
- 3月20日：高亮显示（截止日）
- 3月21日及以后：不显示（已逾期，进入逾期区）
```

#### 单元格内排序

1. 未完成在前，已完成在后
2. 未完成中：优先级 high > medium > low
3. 同优先级按截止时间排序（有时间的在前）
4. 无截止时间的按创建时间排序

### 3.5 区域交互规则

| 操作       | 效果        |
| -------- | --------- |
| 点击逾期区标题  | 折叠/展开逾期区  |
| 点击历史区标题  | 折叠/展开历史区  |
| 点击逾期事项   | 进入事项详情    |
| 点击历史事项   | 进入事项详情    |
| 完成逾期事项   | 从逾期区移入历史区 |
| 取消完成历史事项 | 从历史区移入逾期区 |

## 4. 过去日期处理规则

### 4.1 允许选择过去日期

**规则**：

- 日期选择器不限制只能选未来日期
- 用于补录历史事项或记录逾期任务

### 4.2 过去日期事项归属

| 事项状态 | 截止日期  | 归属区域       |
| ---- | ----- | ---------- |
| 未完成  | < 今天  | 逾期区        |
| 已完成  | < 今天  | 历史区        |
| 未完成  | >= 今天 | 主日期流       |
| 已完成  | >= 今天 | 主日期流（置底显示） |

### 4.3 状态变更后的区域迁移

**场景 1：完成逾期事项**

```
原状态：pending, dueDate = 3天前
原区域：逾期区

操作：点击"完成"

新状态：completed
新区域：历史区
```

**场景 2：取消完成历史事项**

```
原状态：completed, dueDate = 3天前
原区域：历史区

操作：点击"取消完成"

新状态：pending
新区域：逾期区
```

**场景 3：修改逾期事项截止日期到未来**

```
原状态：pending, dueDate = 3天前
原区域：逾期区

操作：修改截止日期为明天

新状态：pending
新区域：主日期流
```

**场景 4：修改未来事项截止日期到过去**

```
原状态：pending, dueDate = 明天
原区域：主日期流

操作：修改截止日期为3天前

新状态：pending
新区域：逾期区
```

## 5. 分类规则

### 5.1 分类 CRUD 规则

#### 创建分类

1. 验证名称唯一性（不区分大小写）
2. 名称长度限制：1-10 字符
3. 颜色默认随机生成
4. `isSystem` 默认为 `false`

#### 编辑分类

1. 可修改：名称、颜色
2. 不可修改：id、isSystem、createdAt
3. 系统内置分类不可编辑

#### 删除分类

1. 系统内置分类不可删除
2. 删除前提示：该分类下的任务将移至"未分类"
3. 确认后执行：
   - 将该分类下所有事项的 `categoryId` 更新为 `uncategorized`
   - 删除分类记录

### 5.2 系统"未分类"分类

| 属性       | 值                 |
| -------- | ----------------- |
| id       | uncategorized（固定） |
| name     | 未分类               |
| color    | #9E9E9E           |
| isSystem | true              |

**约束**：

- 不可删除
- 不可重命名
- 不可修改颜色

**使用场景**：

1. 删除用户分类时，该分类下的所有事项自动迁移到"未分类"
2. AI 解析无法匹配分类时，默认归类到"未分类"
3. 用户创建事项时，分类默认值为"未分类"

## 6. AI 解析规则

### 6.1 文本解析规则

#### 解析范围

| 字段  | 解析方式      | 置信度要求 |
| --- | --------- | ----- |
| 标题  | 提取核心动作+对象 | 高     |
| 日期  | 相对日期/绝对日期 | 中     |
| 时间  | 时间表达式     | 中     |
| 分类  | 关键词匹配     | 低     |
| 优先级 | 紧急程度词汇    | 低     |

#### 相对日期解析

| 输入    | 解析结果       |
| ----- | ---------- |
| 今天    | today      |
| 明天    | today + 1  |
| 后天    | today + 2  |
| 下周一   | 下一个周一      |
| 下周五   | 下一个周五      |
| 下周    | 下周一（默认）    |
| 本周五   | 本周的周五      |
| 周末    | 最近的周六      |
| 月底    | 本月最后一天     |
| 3月15日 | 2026-03-15 |
| 3/15  | 2026-03-15 |

#### 分类关键词匹配

| 关键词               | 推荐分类    |
| ----------------- | ------- |
| 作业/实验/论文/复习/考试/课程 | 个人作业    |
| 小组/汇报/展示/开会/组会/讨论 | 小组作业    |
| 买/采购/下单/补货/购物     | 采购      |
| 其他                | 未分类（默认） |

### 6.2 图片解析规则

#### 解析流程

```
选择图片
    ↓
图片预处理（压缩、格式转换）
    ↓
调用多模态 API
    ↓
┌─────────────┬─────────────┐
↓             ↓             ↓
OCR文字提取   场景识别      多模态理解
    ↓             ↓             ↓
提取日期/时间  推断分类      综合分析
    ↓             ↓             ↓
└─────────────┴─────────────┘
    ↓
生成结构化结果
```

#### 场景识别

| 图片类型 | 识别特征      | 推断分类   |
| ---- | --------- | ------ |
| 课程表  | 表格结构、课程名称 | 个人作业   |
| 会议白板 | 白板背景、待办列表 | 小组作业   |
| 手写便签 | 手写字迹、简短文字 | 根据内容推断 |
| 截图   | 数字界面、截止日期 | 根据内容推断 |

#### 解析失败处理

| 场景     | 处理        |
| ------ | --------- |
| 无法识别文字 | 保留图片，标题为空 |
| 无法识别日期 | 默认今天      |
| 无法识别分类 | 默认"未分类"   |
| API 超时 | 提示重试或手动填写 |

### 6.3 解析结果展示

#### 高置信度（> 80%）

- 直接展示解析结果
- 用户可一键确认或修改

#### 中置信度（50% - 80%）

- 展示解析结果
- 标记为"待确认"
- 高亮显示不确定字段

#### 低置信度（< 50%）

- 仅展示已识别字段
- 其他字段显示默认值
- 提示用户手动完善

## 8. 边界条件处理

### 8.1 数据边界

| 场景     | 处理                |
| ------ | ----------------- |
| 标题为空   | 不允许创建，提示"请输入任务标题" |
| 标题超长   | 实时截断提示，最多100字符    |
| 分类不存在  | 使用"未分类"作为默认值      |
| 日期格式错误 | 默认今天              |
| 时间格式错误 | 忽略时间，仅保留日期        |

### 8.2 业务边界

| 场景          | 处理              |
| ----------- | --------------- |
| 删除有任务的分类    | 提示确认，任务迁移到"未分类" |
| 删除系统"未分类"分类 | 不允许，无删除按钮       |
| 编辑系统分类      | 不允许，无编辑按钮       |
| 无分类时创建任务    | 使用"未分类"作为默认值    |
| 所有分类被删除     | "未分类"始终存在       |

### 8.3 展示边界

| 场景            | 处理            |
| ------------- | ------------- |
| 逾期区无任务        | 不显示逾期区        |
| 历史区无任务        | 不显示历史区        |
| 主日期流无任务       | 显示引导创建提示      |
| 分类过多（> 5个）    | 横向滚动 + 右侧渐变提示 |
| 单元格任务过多（> 5个） | 显示"+N 更多"     |

## 9. 示例场景

### 9.1 创建事项示例

#### 示例 1：创建未来事项

```
输入：
- 标题：完成高数作业
- 分类：个人作业
- 截止日期：明天
- 截止时间：14:00
- 优先级：高

结果：
- status: pending
- displayArea: mainStream
- isOverdue: false
- 展示位置：主日期流"明天"行
```

#### 示例 2：创建过去日期事项（补录）

```
输入：
- 标题：上周的会议记录
- 分类：小组作业
- 截止日期：3天前
- 截止时间：null
- 优先级：中

结果：
- status: pending
- displayArea: overdue
- isOverdue: true
- 展示位置：逾期区
```

#### 示例 3：创建已完成的历史事项

```
输入：
- 标题：上周完成的报告
- 分类：个人作业
- 截止日期：3天前
- 状态：已完成
- 截止时间：null
- 优先级：中

结果：
- status: completed
- displayArea: history
- isOverdue: false（已完成不判断逾期）
```

## 10. 数据一致性保证

### 10.1 写入规则

1. **创建事项**：
   - 自动生成 id、createdAt、updatedAt
   - categoryId 默认为 "uncategorized"
   - status 默认为 "pending"
2. **更新事项**：
   - 自动更新 updatedAt
   - 状态变更时更新 completedAt
3. **删除分类**：
   - 先迁移该分类下所有事项到"未分类"
   - 再删除分类记录

### 10.2 读取规则

1. **逾期判定**：运行时计算，不依赖存储
2. **区域判定**：运行时计算，不依赖存储
3. **排序**：运行时计算，不依赖存储

### 10.3 并发处理

- Alpha 阶段单用户本地存储，无并发问题
- 后续版本需考虑多设备同步冲突处理
