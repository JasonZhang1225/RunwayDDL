## 1. 首页（日期流视图）

### 1.1 页面目标

- 核心功能：展示"日期 × 分类"的任务矩阵
- 用户价值：一眼看清未来多天、多分类的任务分布
- 关键体验：流畅滚动、快速定位、直觉操作

### 1.4 用户操作路径

#### 路径 1：浏览任务

```
进入首页
  ↓
默认展示"今天"起30天的日期流
  ↓
上下滑动浏览不同日期
  ↓
左右滑动查看全部分类
  ↓
点击任务卡片查看详情
```

#### 路径 2：切换视图模式

```
点击左上角 ⇄ 按钮
  ↓
弹出确认对话框："切换为分类纵向视图？"
  ↓
确认后重新渲染
  ↓
原日期列变为横向，分类列变为纵向
  ↓
保持当前筛选条件不变
```

#### 路径 3：筛选分类

```
点击分类标题（如"个人作业"）
  ↓
该分类高亮，其他分类变灰/隐藏
  ↓
矩阵只显示该分类列
  ↓
再次点击分类标题恢复全部显示
```

#### 路径 4：快速定位今天

```
点击"回到今天"按钮
  ↓
平滑滚动动画回到今天所在行
  ↓
今天行高亮闪烁一次（300ms）
```

### 1.5 日期列固定方式

#### 实现方案

**方式：Sticky Header（推荐）**

```dart
// 伪代码示意
custom_scroll_view(
  slivers: [
    // 第一列（日期列）固定
    sliver_to_box_adapter(
      position: sticky_left,
      child: date_column(),
    ),
    // 其余列横向滚动
    sliver_list(
      scroll_direction: horizontal,
      children: category_columns(),
    ),
  ],
)
```

#### 滚动行为

| 操作 | 效果 |
|------|------|
| 上下滑动 | 整个矩阵同步上下滚动，日期列始终可见 |
| 左右滑动 | 仅分类列横向滚动，日期列保持固定 |
| 惯性滚动 | 支持，滚动停止时无对齐要求 |
| 快速滚动 | 显示滚动指示器（Scrollbar） |

### 1.6 分类表头滚动方式

#### 表头结构

```
┌────────┬──────────────────────────────────────┐
│ 日期   │ ← 分类表头横向滚动区域 →              │
│ 标题   │                                      │
├────────┼──────────────────────────────────────┤
│        │                                      │
│ 日期   │ ← 内容区域同步横向滚动 →              │
│ 列     │                                      │
│        │                                      │
└────────┴──────────────────────────────────────┘
```

#### 同步滚动机制

- 表头区域和内容区域使用同一个 ScrollController
- 左右滑动时，表头和内容同步滚动
- 滚动位置保持同步，无错位

### 1.7 单元格任务展示规则

#### 单元格尺寸

| 属性 | 值 | 说明 |
|------|-----|------|
| 宽度 | 120dp | 固定宽度 |
| 最小高度 | 80dp | 无任务时的高度 |
| 最大高度 | 无限制 | 随任务数量扩展 |

#### 任务卡片尺寸

| 属性 | 值 |
|------|-----|
| 宽度 | 112dp（单元格宽度 - 8dp margin） |
| 高度 | 自适应，最小 32dp |
| 外边距 | 4dp |
| 圆角 | 8dp |

#### 任务展示规则

**截止前3天开始显示**（根据 PRD 决策）：

```
假设任务截止日：3月20日

3月17日：开始显示（截止前3天）
3月18日：显示
3月19日：显示
3月20日：高亮显示（截止日）
3月21日：逾期，红色标记
```

**任务排序规则**：

1. 优先级高 → 中 → 低
2. 同优先级按截止时间排序（有时间的在前）
3. 无截止时间的按创建时间排序

**任务卡片内容**：

```
┌─────────────────────────┐
│ ● 任务标题（最多2行）     │  ← 优先级圆点 + 标题
│    14:00               │  ← 截止时间（如有）
└─────────────────────────┘
```

### 1.8 XY 轴互换前后布局对比

#### 模式 A → 模式 B 的映射关系

| 元素 | 模式 A（默认） | 模式 B（互换后） |
|------|---------------|-----------------|
| 纵向主轴 | 日期 | 分类 |
| 横向副轴 | 分类 | 日期 |
| 固定列 | 日期列 | 分类列 |
| 滚动区域 | 分类列 | 日期列 |
| 单元格内容 | 某日期某分类的任务 | 某分类某日期的任务（相同） |

#### 互换动画

- 过渡动画：300ms ease-in-out
- 动画效果：淡出旧布局 → 淡入新布局
- 保持滚动位置百分比（如原来滚动到50%，新布局也在50%）

### 1.9 分类过多/任务过多时的处理

#### 分类过多（> 5个）

**处理方案**：
- 分类列横向滚动（始终）
- 表头显示滚动指示器（右侧渐变遮罩）
- 显示"更多"提示：右侧显示半透明白色遮罩 + 箭头

```
┌────────┬─────────┬─────────┬─────────┬─────────┬───────┐
│ 日期   │ 分类A   │ 分类B   │ 分类C   │ 分类D   │ ▶     │  ← 右侧箭头提示
├────────┼─────────┼─────────┼─────────┼─────────┼───────┤
```

#### 任务过多（单个单元格 > 5个）

**处理方案**：
- 单元格内任务列表可滚动
- 或显示"+3 更多"，点击进入该日期/分类的详情视图
- MVP 推荐：单元格内滚动

```
┌─────────────────┐
│ 任务1           │
│ 任务2           │
│ 任务3           │
│ 任务4           │
│ 任务5           │
│ +3 更多...      │  ← 点击展开或跳转
└─────────────────┘
```

#### 日期范围过大（> 90天）

**处理方案**：
- 使用虚拟列表（Virtual Scroll）
- 只渲染可视区域 ± 3天的内容
- 滚动时动态加载/卸载

### 1.10 空状态

#### 场景 1：某天无任务

```
┌─────────────────┐
│                 │
│   📅            │
│                 │
│  暂无事项        │
│                 │
│                 │
└─────────────────┘
```

- 显示日历图标 + "暂无事项"文字
- 文字颜色：灰色（Colors.grey[400]）
- 点击单元格：打开添加弹窗，预填该日期和分类

#### 场景 2：某分类无任务

- 单元格留空，不显示任何内容
- 保持单元格最小高度

#### 场景 3：全部无任务

```
┌─────────────────────────────────────────┐
│                                         │
│                                         │
│              📋                         │
│                                         │
│         还没有任何任务                   │
│                                         │
│    点击右下角 + 按钮创建第一个任务        │
│                                         │
│                                         │
└─────────────────────────────────────────┘
```

- 全屏居中显示
- 图标：待办清单图标
- 引导文字 + 操作提示

### 1.11 错误状态

#### 错误 1：数据加载失败

```
┌─────────────────────────────────────────┐
│                                         │
│              ⚠️                         │
│                                         │
│         数据加载失败                     │
│                                         │
│      [ 重试 ]                           │
│                                         │
└─────────────────────────────────────────┘
```

- 显示错误图标 + 提示文字
- 提供重试按钮
- 保留本地缓存数据（如有）

#### 错误 2：滚动性能卡顿

- 检测帧率 < 30fps 时自动降级：
  - 隐藏任务卡片动画
  - 减少阴影效果
  - 使用更简单的布局

### 1.12 边界状态

| 场景 | 处理 |
|------|------|
| 分类数量为 0 | 显示引导创建分类的提示 |
| 日期跨月 | 显示月份分隔线 |
| 今天之前日期 | 显示为灰色，不可添加新任务 |
| 逾期任务 | 红色边框 + 红色文字 + 置顶显示 |
| 已完成任务 | 文字划线 + 透明度 50% + 置底 |
| 超长任务标题 | 最多2行，超出显示... |

### 1.13 验收标准

| 检查项 | 标准 |
|--------|------|
| 首页加载 | 30天内数据 2 秒内加载完成 |
| 滚动流畅度 | 60fps，无卡顿 |
| 轴互换 | 切换后 500ms 内完成渲染 |
| 分类过多 | 支持横向滑动，无布局错位 |
| 任务过多 | 单元格内可滚动，或显示"更多" |
| 空状态 | 所有空状态都有引导提示 |
| 响应式 | 适配不同屏幕宽度 |

#### Photo Picker 组件

**功能**：
- 点击"选择图片"：打开系统相册选择器
- 点击"拍照"：打开相机（需权限）
- 支持单张图片（MVP 阶段）

**技术实现**：
- Flutter: `image_picker` 插件
- iOS: `UIImagePickerController` / `PHPickerViewController`
- Android: `PhotoPicker` / `Intent.ACTION_GET_CONTENT`

    ↓             ↓             ↓
