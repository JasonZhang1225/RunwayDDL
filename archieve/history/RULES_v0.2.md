## 文档说明

### 规则优先级体系

| 优先级 | 级别名称 | 说明 |
|--------|----------|------|
| P0 | 核心规则 | 不可违反，违反会导致功能错误 |
| P1 | 重要规则 | 应遵守，特殊情况下可协商调整 |
| P2 | 体验规则 | 建议遵守，影响用户体验但不影响功能 |

### 规则冲突处理原则

1. **P0 规则 > P1 规则 > P2 规则**
2. **显式规则 > 隐式规则**
3. **具体规则 > 通用规则**
4. **若冲突无法解决，以本文档"规则冲突处理"章节为准**

## 1. Item 生命周期

### 1.1 状态流转图

```
┌─────────────┐    创建     ┌─────────────┐
│   无状态    │ ─────────→  │   未完成    │
└─────────────┘             │  (pending)  │
                            └──────┬──────┘
                                   │
                    ┌──────────────┼──────────────┐
                    │              │              │
                    ▼              │              ▼
            ┌─────────────┐        │        ┌─────────────┐
            │   已完成    │ ←──────┘        │    逾期     │
            │ (completed) │   取消完成      │  (overdue)  │
            └─────────────┘                 └─────────────┘
```

### 1.2 状态定义

| 状态 | 标识 | 进入条件 | 退出条件 |
|------|------|----------|----------|
| 未完成 | `pending` | 创建时默认；从已完成取消 | 标记完成；逾期自动转换 |
| 已完成 | `completed` | 用户手动标记完成 | 用户取消完成 |
| 逾期 | `overdue` | 截止日期已过且未完成 | 标记完成；修改截止日期到未来 |

### 1.3 状态转换规则

**规则 1.3.1** [P0] 状态转换只能按图中箭头方向进行，不允许跳跃

**规则 1.3.2** [P0] 逾期是计算状态，不持久化存储，每次查询时实时计算

**规则 1.3.3** [P1] 已完成任务不能再变为逾期，即使截止日期已过

**规则 1.3.4** [P1] 修改任务的截止日期后，重新计算逾期状态

### 1.4 生命周期钩子

| 时机 | 操作 |
|------|------|
| 创建时 | 状态 = pending，记录 createdAt |
| 更新时 | 更新 updatedAt，重新计算逾期状态 |
| 完成时 | 状态 = completed，记录完成时间（扩展字段） |
| 删除时 | 级联删除关联的图片文件 |

## 2. 截止前 3 天显示规则

### 2.1 核心规则

**规则 2.1.1** [P0] 任务在截止前 3 天开始显示，截止日当天及之前均显示

**规则 2.1.2** [P0] 显示窗口计算公式：
```
显示起始日 = 截止日期 - 3天
显示结束日 = 截止日期
```

**规则 2.1.3** [P0] 任务在显示窗口内的每一天都可见，不仅限于截止日期

### 2.2 边界情况

| 场景 | 规则 |
|------|------|
| 截止日 = 今天 | 今天显示（窗口：前3天到今天） |
| 截止日 = 明天 | 今天、明天都显示（窗口：今天到明天） |
| 截止日 = 昨天 | 已逾期，按逾期规则处理 |
| 截止日 = 3天后 | 今天开始显示（窗口：今天到3天后） |
| 截止日 = 4天后 | 今天不显示（窗口从明天开始） |

### 2.3 与逾期规则的关系

**规则 2.3.1** [P0] 逾期任务仍然显示在截止日期单元格，但样式变为逾期样式

**规则 2.3.2** [P1] 逾期任务在截止后的日期不再显示（除非被修改到未来日期）

### 2.4 示例

| 任务截止日 | 今天 | 是否显示 | 说明 |
|------------|------|----------|------|
| 3/20 | 3/17 | ✅ 显示 | 截止前3天 |
| 3/20 | 3/18 | ✅ 显示 | 截止前2天 |
| 3/20 | 3/19 | ✅ 显示 | 截止前1天 |
| 3/20 | 3/20 | ✅ 显示 | 截止日当天 |
| 3/20 | 3/21 | ❌ 不显示 | 已逾期，不跨天显示 |
| 3/20 | 3/16 | ❌ 不显示 | 早于显示窗口 |

## 3. 截止日高亮规则

### 3.1 高亮条件

**规则 3.1.1** [P0] 同时满足以下条件时，任务在截止日高亮显示：
1. 当前日期 = 任务截止日期
2. 任务状态 = 未完成

**规则 3.1.2** [P0] 已完成任务在截止日不高亮

**规则 3.1.3** [P0] 逾期任务（截止日期已过）显示逾期样式，不显示截止日高亮

### 3.2 高亮样式

### 3.3 与显示窗口的关系

**规则 3.3.1** [P1] 任务在显示窗口内都可见，但只有截止日当天高亮

**规则 3.3.2** [P1] 截止前 1-3 天的任务正常显示，不高亮

## 4. 逾期判定规则

### 4.1 核心判定公式

**规则 4.1.1** [P0] 逾期判定条件：
```dart
isOverdue = (status == pending) && (dueDate < today)
```

**规则 4.1.2** [P0] `today` 定义为当前日期的 00:00:00（不含时间）

**规则 4.1.3** [P0] 比较时不考虑 `dueTime`，只看日期部分

### 4.2 逾期样式

| 元素 | 逾期样式 |
|------|----------|
| 任务卡片边框 | 红色（#F44336） |
| 任务标题 | 红色，加粗 |
| 优先级圆点 | 保持原色 |
| 背景 | 浅红色（#FFEBEE） |

### 4.3 逾期任务的排序

**规则 4.3.1** [P1] 逾期任务在单元格内置顶显示（排在最前面）

**规则 4.3.2** [P1] 多个逾期任务按原排序规则排列

### 4.4 逾期与完成的互斥

**规则 4.4.1** [P0] 已完成任务永远不会被判定为逾期

**规则 4.4.2** [P0] 逾期任务标记完成后，逾期状态立即解除

### 4.5 示例

| 截止日期 | 当前日期 | 状态 | 是否逾期 | 说明 |
|----------|----------|------|----------|------|
| 3/20 | 3/21 | 未完成 | ✅ 逾期 | 已过截止日 |
| 3/20 | 3/20 | 未完成 | ❌ 不逾期 | 今天不算逾期 |
| 3/20 | 3/19 | 未完成 | ❌ 不逾期 | 截止前 |
| 3/20 | 3/21 | 已完成 | ❌ 不逾期 | 已完成不逾期 |
| 3/20 14:00 | 3/20 15:00 | 未完成 | ❌ 不逾期 | 只看日期，不看时间 |

## 5. 完成状态规则

### 5.1 完成操作

**规则 5.1.1** [P0] 用户可通过以下方式标记完成：
- 首页长按任务卡片 → 选择"完成"
- 任务详情页点击"标记完成"按钮
- 任务卡片左侧复选框（如设计支持）

**规则 5.1.2** [P0] 标记完成后，任务状态立即变为 `completed`

### 5.2 完成样式

| 元素 | 完成样式 |
|------|----------|
| 任务标题 | 中划线，颜色变淡（透明度 50%） |
| 任务卡片 | 背景变淡（透明度 70%） |
| 优先级圆点 | 变为灰色 |

### 5.3 完成任务的排序

**规则 5.3.1** [P0] 已完成任务排在单元格最底部

**规则 5.3.2** [P1] 已完成任务之间按完成时间倒序（最近完成的在前）

### 5.4 取消完成

**规则 5.4.1** [P0] 用户可取消完成状态，任务回到 `pending`

**规则 5.4.2** [P0] 取消完成后，重新计算逾期状态

**规则 5.4.3** [P1] 若取消完成时已逾期，立即显示逾期样式

## 6. 首页排序规则

### 6.1 单元格内排序优先级

**规则 6.1.1** [P0] 单元格内任务按以下优先级排序（高到低）：

| 优先级 | 条件 | 排序规则 |
|--------|------|----------|
| 1 | 逾期 + 未完成 | 置顶，按逾期天数倒序（逾期越久越前） |
| 2 | 未完成 + 高优先级 | 按截止时间升序（早的在前） |
| 3 | 未完成 + 中优先级 | 按截止时间升序 |
| 4 | 未完成 + 低优先级 | 按截止时间升序 |
| 5 | 已完成 | 置底，按完成时间倒序 |

### 6.2 同优先级内排序

**规则 6.2.1** [P1] 同优先级内，有截止时间的排在无截止时间的前面

**规则 6.2.2** [P1] 都有截止时间时，按时间升序（早的在前）

**规则 6.2.3** [P2] 都无截止时间时，按创建时间升序（早的在前）

### 6.3 排序算法伪代码

```dart
int compareItems(Item a, Item b) {
  // 1. 已完成 vs 未完成
  if (a.status != b.status) {
    return a.status == pending ? -1 : 1;
  }
  
  // 2. 逾期 vs 未逾期（都未完成时）
  if (a.isOverdue != b.isOverdue) {
    return a.isOverdue ? -1 : 1;
  }
  
  // 3. 优先级
  final priorityOrder = {high: 0, medium: 1, low: 2};
  if (a.priority != b.priority) {
    return priorityOrder[a.priority]!.compareTo(priorityOrder[b.priority]!);
  }
  
  // 4. 截止时间
  if (a.dueTime != null && b.dueTime == null) return -1;
  if (a.dueTime == null && b.dueTime != null) return 1;
  if (a.dueTime != null && b.dueTime != null) {
    return a.dueTime!.inMinutes.compareTo(b.dueTime!.inMinutes);
  }
  
  // 5. 创建时间
  return a.createdAt.compareTo(b.createdAt);
}
```

## 7. 分类筛选规则

### 7.1 筛选触发

**规则 7.1.1** [P0] 点击首页分类表头，触发分类筛选

**规则 7.1.2** [P0] 再次点击同一分类，取消筛选，恢复显示全部分类

### 7.2 筛选效果

**规则 7.2.1** [P0] 筛选后，矩阵只显示被选中的分类列

**规则 7.2.2** [P0] 被选中的分类表头高亮显示（背景色加深）

**规则 7.2.3** [P1] 筛选状态下，"回到今天"按钮仍然有效

### 7.3 筛选与其他规则的交互

**规则 7.3.1** [P0] 筛选不影响任务的显示窗口规则（截止前3天）

**规则 7.3.2** [P0] 筛选不影响任务的排序规则

**规则 7.3.3** [P1] 筛选状态下添加新任务，若新任务属于被筛选分类，立即显示

### 7.4 空状态

**规则 7.4.1** [P1] 筛选后若无任务，显示"该分类暂无任务"提示

## 8. 时间范围筛选规则

### 8.1 筛选选项

| 选项 | 范围定义 |
|------|----------|
| 今天 | 今天 00:00:00 至 今天 23:59:59 |
| 近 7 天 | 今天 至 今天+6天 |
| 近 30 天 | 今天 至 今天+29天 |
| 全部 | 今天 至 未来无限 |

### 8.2 默认设置

**规则 8.2.1** [P1] 默认选中"近 30 天"

**规则 8.2.2** [P2] 用户上次选择应被记忆（当前会话内）

### 8.3 筛选效果

**规则 8.3.1** [P0] 只显示在筛选范围内的日期行

**规则 8.3.2** [P0] 任务的显示仍遵循"截止前3天"规则，但只显示在筛选范围内的日期

**规则 8.3.3** [P1] 若任务的显示窗口与筛选范围部分重叠，只显示重叠部分

### 8.4 示例

| 任务截止日 | 筛选范围 | 显示在哪些日期 |

| 3/20 | 近7天（3/14-3/20） | 3/17, 3/18, 3/19, 3/20 |

| 3/18 | 近7天（3/14-3/20） | 3/15, 3/16, 3/17, 3/18 |

## 9. XY 轴互换后规则变化

### 9.1 不变规则

以下规则在 XY 轴互换后**完全不变**：

| 规则 | 说明 |
|------|------|
| 截止前 3 天显示规则 | 任务仍按截止前3天显示 |
| 截止日高亮规则 | 截止日仍高亮 |
| 逾期判定规则 | 逾期判定逻辑不变 |
| 完成状态规则 | 完成/未完成逻辑不变 |
| 任务排序规则 | 单元格内排序规则不变 |

### 9.2 变化规则

以下规则在 XY 轴互换后**发生变化**：

| 规则 | 默认模式 | 互换模式 |
|------|----------|----------|
| 固定列 | 日期列固定 | 分类列固定 |
| 横向滚动内容 | 分类列 | 日期列 |
| 表头内容 | 分类名称 | 日期 |
| 主标签 | 日期（纵向） | 分类（纵向） |

### 9.3 筛选规则

**规则 9.3.1** [P0] XY 轴互换后，分类筛选仍然有效

**规则 9.3.2** [P0] 互换模式下点击分类表头（此时在左侧固定列），筛选该分类

**规则 9.3.3** [P1] 时间范围筛选在两种模式下都有效

### 9.4 数据组织

**规则 9.4.1** [P0] 互换不改变底层数据，只改变视图组织方式

**规则 9.4.2** [P0] 同一任务在两种模式下显示的内容完全一致

## 11. 规则冲突处理

### 11.1 已知冲突场景

#### 冲突 1：逾期 vs 截止前3天显示

**场景**：任务已逾期，但按截止前3天规则应该显示

**冲突**：
- 规则 2.1.1：截止前3天开始显示
- 规则 2.3.2：逾期任务在截止后的日期不再显示

**裁决** [P0]：
- 逾期任务只显示在截止日期当天，不显示在截止后的日期
- 逾期任务在截止前3天到截止日之间的显示不受影响

**示例**：
- 截止日 3/15，今天 3/17
- 显示：只在 3/15 显示（逾期样式）
- 不显示：3/16、3/17（已过显示窗口）

#### 冲突 2：完成 vs 高亮

**场景**：任务已完成，但今天是截止日

**冲突**：
- 规则 3.1.1：截止日高亮
- 规则 3.1.2：已完成任务不高亮

**裁决** [P0]：
- 已完成任务在截止日不高亮
- 高亮只针对未完成的任务

#### 冲突 3：筛选 vs 显示窗口

**场景**：分类筛选后，某任务的显示窗口部分在筛选范围内

**冲突**：
- 规则 7.3.1：筛选不影响显示窗口规则
- 规则 8.3.2：只显示在筛选范围内的日期

**裁决** [P0]：
- 先应用显示窗口规则，确定任务应显示在哪些日期
- 再应用筛选规则，只保留在筛选范围内的日期
- 最终显示 = 显示窗口 ∩ 筛选范围

### 11.2 冲突解决优先级

当规则冲突时，按以下优先级裁决：

1. **状态规则 > 显示规则**
   - 已完成/逾期状态优先于显示窗口

2. **时间规则 > 分类规则**
   - 日期相关的规则优先于分类筛选

3. **核心规则 > 体验规则**
   - P0 规则优先于 P1/P2 规则
