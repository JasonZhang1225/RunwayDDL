# RunwayDDL 产品文档 PRD v0.2

**版本日期**：2026-03-14\
**适用范围**：MVP 阶段（v0.2.0 - v0.2.x）

***

## 1. 产品概述

### 1.1 一句话定义

RunwayDDL 是一个以**日期纵向流**为主轴、以**分类横向跑道**并列展示任务的管理工具，支持 AI 辅助快速录入。

### 1.2 核心验证目标（MVP 阶段）

1. 用户是否理解并接受"日期 + 分类"的矩阵视图
2. 用户是否愿意在单一视图中长期维护任务
3. AI 解析是否能降低录入成本（而非增加纠错成本）

### 1.3 目标用户画像

**核心用户**：大学生、研究生、轻量协作团队成员

**共性需求**：

- 同时面对"什么时候截止"和"这件事属于哪一类"两个维度
- 需要快速查看某天不同类型任务的负载情况
- 希望通过自然语言快速记录任务

## 2. 核心概念

### 2.1 日期流（Date Stream）

以日期为纵向主轴的连续列表，从"今天"开始向下延伸。

**展示规则**：

- 默认展示从今天起未来 30 天的日期
- 支持"近 7 天 / 近 30 天 / 全部"筛选
- 每个日期节点占据一行

### 2.2 跑道（Lane）

横向排列的分类列，每个分类对应一列。

**展示规则**：

- 分类数量无硬性限制，动态增加
- 分类过多时支持横向滑动
- 支持 XY 轴互换：点击左上角按钮，可将日期和分类的展示维度互换

### 2.3 事项（Item）

系统中的基础任务单元。

**字段定义**：

- 标题（必填）
- 所属分类（必填）
- 截止日期（必填）
- 截止时间（可选）
- 优先级：高 / 中 / 低（默认中）
- 状态：未完成 / 已完成
- 描述（可选）

**跨天展示规则**：

- 截止前 3 天开始在该日期显示该任务
- 截止日当天高亮显示
- 逾期后标记为红色并置顶

### 2.4 概念关系图

```
┌─────────────────────────────────────────────────┐
│                  日期流视图                       │
├─────────────────────────────────────────────────┤
│  日期轴 │ 分类A  │ 分类B  │ 分类C  │ ...        │
├─────────┼────────┼────────┼────────┼────────────┤
│  今天   │ 任务1  │ 任务2  │   -    │            │
│  明天   │   -    │ 任务3  │ 任务4  │            │
│  后天   │ 任务5  │   -    │   -    │            │
│  ...    │  ...   │  ...   │  ...   │            │
└─────────────────────────────────────────────────┘
```

## 4. 功能规格

### 4.1 首页（日期流视图）

#### 布局规格

**顶部区域**：

- 左上角：XY 轴互换按钮
- 中间：当前视图范围（今天 / 7天 / 30天 / 全部）
- 右上角：搜索按钮（MVP 可 placeholder）

**主内容区域**：

- 日期列：显示日期和星期，随滚动自然排列
- 分类列：横向排列，支持左右滑动
- 单元格：显示该日期该分类下的任务列表

**底部区域**：

- 悬浮"+"按钮：打开添加事项弹窗
- "回到今天"按钮：快速滚动回今天

#### 交互规则

| 操作         | 响应               |
| ---------- | ---------------- |
| 上下滑动       | 滚动浏览日期           |
| 左右滑动       | 滑动查看更多分类         |
| 点击 XY 互换按钮 | 日期和分类维度互换        |
| 点击事项       | 进入事项详情           |
| 长按事项       | 弹出快捷菜单（完成/编辑/删除） |
| 点击分类标题     | 筛选只显示该分类（再次点击取消） |

#### 空状态

- **某天无任务**：显示"暂无事项"占位符
- **某分类无任务**：单元格留空
- **全部无任务**：展示引导创建第一个任务的提示

### 4.2 分类管理

#### CRUD 规格

**创建分类**：

- 输入：分类名称（必填，最多 10 字）、颜色（选填，默认随机）
- 限制：无数量上限

**编辑分类**：

- 可修改：名称、颜色
- 不可修改：创建时间

**删除分类**：

- 删除前提示：该分类下的任务将变为"未分类"
- 支持批量转移：可将分类 A 的任务全部转移到分类 B

**查询分类**：

- 首页展示所有分类
- 支持按名称搜索（MVP 可选）

#### 约束规则

- 分类名称唯一（不区分大小写）
- 默认提供 3 个示例分类：个人作业、小组作业、其他
- 分类颜色用于：表头背景、任务卡片边框、截止高亮

### 4.3 事项管理

#### 手动创建

**入口**：首页点击"+"按钮

**表单字段**：

| 字段   | 类型   | 必填 | 默认值   |
| ---- | ---- | -- | ----- |
| 标题   | 文本   | 是  | -     |
| 分类   | 单选   | 是  | 第一个分类 |
| 截止日期 | 日期选择 | 是  | 今天    |
| 截止时间 | 时间选择 | 否  | 空     |
| 优先级  | 单选   | 否  | 中     |
| 描述   | 多行文本 | 否  | 空     |

#### 快捷添加

**输入形式**：自然语言输入框

**示例输入**：

- "明天交英语作业"
- "周五晚上和小组讨论展示"
- "下周买打印纸和胶带"
- "4月2日交实验报告"

**解析后展示**：

- 已识别标题：英语作业
- 已识别日期：明天（3月15日）
- 推荐分类：个人作业
- 用户可一键确认或修改任意字段

#### 完成/编辑/删除

**完成**：

- 点击任务左侧复选框
- 已完成任务：文字划线、颜色变淡、自动置底

**编辑**：

- 进入详情页或长按选择"编辑"
- 可修改所有字段

**删除**：

- 长按选择"删除"
- 二次确认后删除

### 4.4 AI 功能

#### 解析范围

**文本输入识别**：

- 事项标题
- 日期（相对或绝对）
- 分类建议
- 截止时间（可选）
- 优先级暗示（"紧急"→高优先级，可选）

**图片输入识别（多模态）**：

- 支持从相册选择图片或拍照上传
- 识别图片中的文字内容（OCR）
- 识别图片中的日期、时间信息
- 识别图片场景推断任务类型（如：课程表、会议白板、手写便签）
- 结合图片内容生成任务标题和描述

**输入方式**：

- 文本输入：自然语言描述
- 图片输入：Photo Picker 选择图片或相机拍照

#### 推荐逻辑

**文本输入处理**：

**第一层：关键词匹配**

- "作业/实验/论文/复习" → 个人作业
- "小组/汇报/展示/开会/组会" → 小组作业
- "买/采购/下单/补货" → 采购

**第二层：模型判断**

- 对规则不明确的任务调用国产 LLM
- 输出：推荐分类 + 置信度

**图片输入处理**：

**第一层：OCR 文字提取**

- 提取图片中的所有文字内容
- 识别日期、时间关键词（如：3月15日、周五、下周二等）

**第二层：视觉场景识别**

- 识别图片类型：课程表、会议白板、手写便签、截图、文档等
- 根据场景类型推断任务分类

**第三层：多模态模型理解**

- 调用支持多模态的国产模型（如通义千问 VL、文心一言多模态版）
- 综合分析图片内容和 OCR 结果
- 生成结构化任务信息

**第三层：用户反馈（记录，MVP 阶段不实现学习）**

- 记录用户修正行为
- 为后续个性化学习积累数据

#### 结果呈现

**文本解析成功**：

- 展示解析结果预览
- 用户一键确认或展开修改

**图片解析成功**：

- 展示原图缩略图
- 展示识别的文字内容（可编辑）
- 展示解析结果预览
- 用户可修改任意字段

**解析失败**：

- 文本输入：仅提取标题（原文），其他字段默认
- 图片输入：保留图片附件，标题为空，日期默认今天，分类默认"其他"
- 提示用户手动完善信息

### 4.5 筛选功能

**时间范围筛选**：

- 今天
- 近 7 天
- 近 30 天
- 全部未来

**分类筛选**：

- 点击分类标题：只显示该分类
- 再次点击：取消筛选，显示全部

**状态筛选**（MVP 可选）：

- 全部
- 未完成
- 已完成

## 5. 数据模型

### 5.1 Category 表

```json
{
  "id": "uuid",
  "name": "string (max 10 chars)",
  "color": "string (hex color)",
  "sort_order": "integer",
  "created_at": "timestamp",
  "updated_at": "timestamp"
}
```

### 5.2 Item 表

```json
{
  "id": "uuid",
  "title": "string (max 100 chars)",
  "description": "string (optional)",
  "category_id": "uuid",
  "due_date": "date",
  "due_time": "time (optional)",
  "priority": "enum: high/medium/low",
  "status": "enum: pending/completed",
  "image_path": "string (optional)",
  "created_at": "timestamp",
  "updated_at": "timestamp"
}
```

**字段说明**：
- `image_path`: 图片附件本地存储路径（快捷添加时通过图片识别创建的任务可保留原图）

### 5.3 AI\_Log 表（可选，用于后续优化）

```json
{
  "id": "uuid",
  "item_id": "uuid",
  "raw_input": "string",
  "parsed_title": "string",
  "parsed_date": "date",
  "predicted_category_id": "uuid",
  "confidence": "float",
  "user_corrected": "boolean",
  "created_at": "timestamp"
}
```

## 7. 技术约束

### 7.1 技术栈

- **框架**：Flutter
- **语言**：Dart
- **状态管理**：Provider / Riverpod（待定）
- **本地存储**：Hive / SQLite
- **AI 接入**：国产多模态 LLM API（通义千问 VL / 文心一言多模态版）
- **图片处理**：`image_picker` 插件

### 7.2 数据存储方案

**MVP 阶段**：纯本地存储

- 使用 Hive 或 SQLite 存储
- 数据文件位置：应用沙盒目录
- 暂不支持云端备份
- 暂不支持跨设备同步

**数据安全**：

- 应用卸载时数据丢失（需用户知晓）
- 后续版本考虑导出/导入功能

### 7.3 AI 接入方案

**文本解析接口**：

- 首选：通义千问 / 文心一言
- 备选：智谱 GLM / 讯飞星火

**多模态图片解析接口**：

- 首选：通义千问 VL（支持图文理解）
- 备选：文心一言多模态版

**调用方式**：

- 服务端代理（保护 API Key）
- 或前端直接调用（MVP 可接受，需配置 CORS）

**文本解析 Prompt 模板**：

```
请从以下自然语言描述中提取任务信息，返回 JSON 格式：

输入：{user_input}

返回格式：
{
  "title": "任务标题",
  "date": "YYYY-MM-DD 或相对日期描述",
  "time": "HH:MM 或 null",
  "category_hint": "分类关键词",
  "priority": "high/medium/low"
}
```

**图片解析 Prompt 模板**：

```
请分析这张图片，提取其中的任务信息：

1. 如果是课程表：提取课程名称、时间、地点
2. 如果是会议白板：提取会议主题、时间、待办事项
3. 如果是手写便签：提取文字内容中的任务信息
4. 如果是截图：提取其中的截止日期、任务描述

返回 JSON 格式：
{
  "title": "任务标题",
  "date": "YYYY-MM-DD",
  "time": "HH:MM 或 null",
  "category_hint": "分类关键词",
  "priority": "high/medium/low",
  "ocr_text": "图片中识别的原始文字"
}
```

## 8. 非功能需求

### 8.1 性能要求

- 首页加载：30 天内数据秒开
- 滚动流畅：60fps，无卡顿
- AI 解析：3 秒内返回结果
- 添加任务：1 秒内完成保存并刷新视图

### 8.2 兼容性要求

- **iOS**：iOS 14.0+
- **Android**：Android 8.0+
- 适配手机和平板（响应式布局）

### 8.3 安全要求

- API Key 不硬编码在客户端（使用环境变量或代理）
- 用户数据不传输到第三方（除 AI 解析的必要文本）

### 10.2 已知技术债务

1. **本地存储限制**：MVP 使用本地存储，后续必须迁移到云端
2. **AI 准确率**：依赖国产模型，准确率可能波动，需要兜底策略
3. **性能瓶颈**：分类数量过多时可能影响渲染性能，需虚拟列表优化
