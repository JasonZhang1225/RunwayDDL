```markdown
### 4.1 首页（日期流视图）

#### 布局规格

**整体结构**：双层展示

**上层：固定历史/逾期区**

| 区域 | 默认状态 | 内容 | 排序规则 |
|------|----------|------|----------|
| 逾期区 | 展开 | 未完成 && dueDate < today | 按逾期天数倒序（逾期越久越前） |
| 历史区 | 折叠 | 已完成 && dueDate < today | 按完成时间倒序 |

**下层：主日期流矩阵**

- 起始日期：今天
- 展示范围：固定未来 30 天
- Alpha 阶段不支持时间范围筛选

**顶部区域**：

- 左上角：XY 轴互换按钮（Alpha 后段实现）
- 中间：页面标题 "RunwayDDL"
- 右上角：搜索按钮（Beta 功能）

**主内容区域**：

- 日期列：显示日期和星期
- 分类列：横向排列，支持左右滑动
- 单元格：显示该日期该分类下的任务列表

**底部区域**：

- 悬浮"+"按钮：打开添加事项弹窗
- "回到今天"按钮：快速滚动回今天

#### 交互规则

| 操作 | 响应 |
|------|------|
| 上下滑动 | 滚动浏览日期（主日期流区域） |
| 左右滑动 | 滑动查看更多分类 |
| 点击事项 | 进入事项详情 |
| 长按事项 | 弹出快捷菜单（完成/编辑/删除） |
| 点击逾期区事项 | 进入事项详情 |
| 展开历史区 | 显示所有历史已完成事项 |
| 折叠历史区 | 隐藏历史事项 |

#### 空状态

- **逾期区无任务**：不显示逾期区
- **历史区无任务**：不显示历史区
- **主日期流无任务**：显示引导创建提示
- **全部无任务**：展示引导创建第一个任务的提示
```

```markdown
#### 系统内置分类

**"未分类"分类**：

| 属性 | 值 |
|------|-----|
| ID | uncategorized（固定） |
| 名称 | 未分类 |
| 颜色 | 默认灰色 #9E9E9E |
| 属性 | 系统内置、不可删除、不可重命名、不可修改颜色 |

**使用场景**：

1. 删除用户分类时，该分类下的所有事项自动迁移到"未分类"
2. AI 解析无法匹配分类时，默认归类到"未分类"
3. 用户创建事项时，必须选择分类（不能为空）

**删除分类规则**：

- 删除前提示："确定删除'XX'？该分类下的 N 个任务将移至'未分类'"
- 用户确认后执行删除 + 批量迁移
- 可选：提供"转移到其他分类"选项，但默认兜底为"未分类"
```

#### 过去日期事项处理

- 日期选择器不限制只能选未来日期
- 用于补录历史事项或记录逾期任务

**原内容位置**：PRODUCT_v0.2.md 第 364-380 行

**修订后内容**：

```markdown
### 5.2 Item 表

```json
{
  "id": "uuid",
  "title": "string (max 100 chars)",
  "description": "string (optional)",
  "category_id": "uuid",
  "due_date": "date",
  "due_time": "time (optional)",
  "priority": "enum: high/medium/low",
  "status": "enum: pending/completed",
  "image_path": "string (optional)",
  "created_at": "timestamp",
  "updated_at": "timestamp"
}
```

**字段说明**：

- `status`：只有两种值 `pending`（未完成）和 `completed`（已完成）
- **不存在 `overdue` 字段**：逾期状态为运行时计算，不持久化
- 逾期判定公式：`isOverdue = (status == pending) && (due_date < today)`
- `image_path`：图片附件本地存储路径

**计算属性（不存储）**：

| 属性 | 计算方式 |
|------|----------|
| isOverdue | `(status == pending) && (due_date < today)` |
| displayArea | 根据 due_date 和 status 判断归属区域 |
```

### 新增：4.6 历史/逾期区规则

```markdown
### 4.6 历史/逾期区规则

#### 逾期区

**显示条件**：
- 存在未完成且 dueDate < today 的事项

**默认状态**：展开

**排序规则**：
1. 按逾期天数倒序（逾期越久越前）
2. 同逾期天数按优先级排序
3. 同优先级按截止时间排序

**样式**：
- 逾期事项卡片：红色边框 + 红色标题
- 显示逾期天数标签："逾期 N 天"

#### 历史区

**显示条件**：
- 存在已完成且 dueDate < today 的事项

**默认状态**：折叠

**展开后排序规则**：
- 按完成时间倒序（最近完成的在前）

**样式**：
- 历史事项卡片：中划线 + 透明度 50%
- 显示完成时间

#### 区域交互

| 操作 | 效果 |
|------|------|
| 点击逾期区标题 | 折叠/展开逾期区 |
| 点击历史区标题 | 折叠/展开历史区 |
| 点击逾期事项 | 进入事项详情 |
| 点击历史事项 | 进入事项详情 |
| 完成逾期事项 | 从逾期区移入历史区 |
```
