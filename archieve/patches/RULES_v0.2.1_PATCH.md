### 1.2 状态定义

| 状态 | 标识 | 存储位置 | 进入条件 | 退出条件 |
|------|------|----------|----------|----------|
| 未完成 | `pending` | 数据库持久化 | 创建时默认；从已完成取消 | 标记完成 |
| 已完成 | `completed` | 数据库持久化 | 用户手动标记完成 | 用户取消完成 |
| 逾期 | `overdue` | **运行时计算** | (status == pending) && (dueDate < today) | 标记完成；修改截止日期到未来 |

**规则 1.2.1** [P0] `overdue` 是**纯计算状态**，不持久化到数据库

**规则 1.2.2** [P0] 每次查询事项时，实时计算逾期状态

**规则 1.2.3** [P0] 数据库中 `status` 字段只有两种值：`pending` 或 `completed`

**规则 1.2.4** [P0] 禁止在数据模型中添加 `overdue` 字段
```

## 2.5 历史/逾期区规则

### 2.5.1 区域定义

| 区域 | 显示条件 | 默认状态 | 内容 |
|------|----------|----------|------|
| 逾期区 | 存在未完成 && dueDate < today 的事项 | 展开 | 所有逾期未完成事项 |
| 历史区 | 存在已完成 && dueDate < today 的事项 | 折叠 | 所有历史已完成事项 |

### 2.5.2 事项归属判定

**规则 2.5.1** [P0] 事项归属判定公式：

```
if (dueDate < today) {
  if (status == completed) {
    归属 = 历史区
  } else {
    归属 = 逾期区
  }
} else {
  归属 = 主日期流
}
```

**规则 2.5.2** [P0] 逾期区事项不在主日期流中显示

**规则 2.5.3** [P0] 历史区事项不在主日期流中显示

**规则 2.5.4** [P1] 逾期区为空时不显示该区域

**规则 2.5.5** [P1] 历史区为空时不显示该区域

### 2.5.3 逾期区排序规则

**规则 2.5.6** [P0] 逾期区事项按以下优先级排序：

| 优先级 | 条件 | 排序规则 |
|--------|------|----------|
| 1 | 逾期天数多 | 逾期天数倒序（逾期越久越前） |
| 2 | 同逾期天数 | 优先级高 → 中 → 低 |
| 3 | 同优先级 | 截止时间早的在前 |
| 4 | 同截止时间 | 创建时间早的在前 |

**规则 2.5.7** [P1] 逾期天数计算公式：
```
逾期天数 = today - dueDate（天数差）
```

### 2.5.4 历史区排序规则

**规则 2.5.8** [P0] 历史区事项按完成时间倒序（最近完成的在前）

**规则 2.5.9** [P1] 历史区可折叠/展开，状态在当前会话内保持

### 2.5.5 状态变更后的区域迁移

**规则 2.5.10** [P0] 状态变更触发区域迁移：

| 操作 | 原区域 | 新区域 | 迁移时机 |
|------|--------|--------|----------|
| 完成逾期事项 | 逾期区 | 历史区 | 立即迁移 |
| 取消完成历史事项 | 历史区 | 逾期区 | 立即迁移 |
| 修改截止日期到未来（逾期事项） | 逾期区 | 主日期流 | 立即迁移 |
| 修改截止日期到过去（未完成） | 主日期流 | 逾期区 | 立即迁移 |
| 修改截止日期到过去（已完成） | 主日期流 | 历史区 | 立即迁移 |

### 2.5.6 区域样式

| 区域 | 事项样式 |
|------|----------|
| 逾期区 | 红色边框 + 红色标题 + 逾期天数标签 |
| 历史区 | 中划线 + 透明度 50% + 完成时间 |
```

## 4. 逾期判定规则

### 4.1 核心判定公式

**规则 4.1.1** [P0] 逾期判定条件：
```dart
isOverdue = (status == pending) && (dueDate < today)
```

**规则 4.1.2** [P0] `today` 定义为当前日期的 00:00:00（不含时间）

**规则 4.1.3** [P0] 比较时不考虑 `dueTime`，只看日期部分

**规则 4.1.4** [P0] **逾期状态不持久化**，每次查询时实时计算

### 4.2 逾期事项的显示位置

**规则 4.2.1** [P0] 逾期事项显示在首页"逾期区"，不在主日期流中显示

**规则 4.2.2** [P0] 逾期事项不占用过去日期的单元格

**规则 4.2.3** [P1] 逾期区默认展开，用户可手动折叠

### 4.3 逾期样式

| 元素 | 逾期样式 |
|------|----------|
| 任务卡片边框 | 红色（#F44336） |
| 任务标题 | 红色，加粗 |
| 逾期标签 | "逾期 N 天" |
| 背景 | 浅红色（#FFEBEE） |

### 4.4 逾期与完成的互斥

**规则 4.4.1** [P0] 已完成任务永远不会被判定为逾期

**规则 4.4.2** [P0] 逾期任务标记完成后，立即从逾期区移入历史区

### 4.5 示例

| 截止日期 | 当前日期 | 状态 | 是否逾期 | 显示位置 |
|----------|----------|------|----------|----------|
| 3/15 | 3/17 | 未完成 | ✅ 逾期 | 逾期区 |
| 3/15 | 3/17 | 已完成 | ❌ 不逾期 | 历史区 |
| 3/17 | 3/17 | 未完成 | ❌ 不逾期 | 主日期流（今天） |
| 3/20 | 3/17 | 未完成 | ❌ 不逾期 | 主日期流 |
```
